ONDRIS

A proof-of-work chain where every hash has to touch a wide, shifting block of memory. That's the entire design: memory is the one resource a graphics card already has in abundance — and the one an ASIC would have to buy from scratch, at a cost its speed can't make back.

How it works

Three steps, and none of them are a shortcut.

OndrisHash doesn't invent a new cryptographic primitive — it combines BLAKE3, already audited and standardized, into an original arrangement of memory that's expensive to fake.

01

Epoch dataset

Every 2,048 blocks the network derives a new dataset — tens of megabytes generated from the hash of a block that's already been mined. Nobody can precompute it ahead of time, because it doesn't exist until the chain writes it.

02

Dataset mixing

Mining reads 64 pseudo-random slices out of that dataset per attempt and folds each one in with a cheap mix step. Every attempt is dominated by real, scattered memory reads across tens of megabytes — not by raw arithmetic a single fast core could win at alone.

03

Validation

The result only counts if it comes out below the network's current target — a plain 256-bit comparison any node can check in microseconds, whether or not it owns a GPU.

Why this favors a GPU

The bottleneck is bandwidth, not brains.

What the algorithm rewards

  • A wide memory bus moving many gigabytes per second — ordinary on a graphics card.
  • Thousands of threads that can each fire off their own random dataset reads in parallel.
  • Hardware you already own and can repurpose the moment you stop mining.

What it resists

  • A dedicated ASIC would need to embed the same fast VRAM a GPU already ships with.
  • That erases most of an ASIC's usual cost and power advantage before it even starts hashing.
  • A CPU can still verify a block in the time it takes to read this sentence — it just can't mine competitively.

Get started

Run the node, then point a GPU at it.

testnet
$ cargo build --release --workspace

$ cargo run --release --bin ondris-node -- --peer 51.89.226.44:30303

$ cargo run --release --bin ondris-wallet -- new --out my-wallet.json

$ cargo run --release --bin ondris-miner -- --node http://127.0.0.1:8080 --address <your-address> --threads 4

Needs Rust (edition 2021, 1.96+). The commands above build every binary, join the public testnet seed, create a wallet with an encrypted keystore, and start mining with all available CPU threads. There's also an OpenCL GPU miner (ondris-miner-gpu) — run its self-test subcommand before pointing it at real hardware; it's correctness-validated and measures ~13M H/s on an RTX 4070 Super.

Full walkthrough, RPC reference, and known limitations are in the README and architecture notes.
